### SSH lessons learned (bash + SSH array pitfall)

When piping bash scripts over SSH:
- Storing an SSH command as a **scalar variable** (`SSH_CMD="ssh ..."`) and then
  executing it as `"$SSH_CMD arg"` treats the whole string as a **single filename** — fails
  with "No such file or directory"
- **Fix:** use a bash **array**: `SSH_CMD=(ssh -o ... opts ... user@host)` and execute
  as `"${SSH_CMD[@]}"` — wordsplitting works correctly

Also learned:
- `bash -s --local` — use `--` separator so `--local` is a script argument, not a bash flag
- `set -e` + pipelines = PIPESTATUS trap; always use `|| true` on pipeline chains
- Subshells in `while read` loops swallow `return` — use `for` loops with `|| true` instead
- `run_cmd` must use `|| true` on every command call to avoid `set -e` aborting on
  legitimate non-zero exits (journalctl, chronyc, systemctl, etc.)
- `journalctl -u ssh -u sshd` is invalid syntax — run two separate invocations
- `grep -c` returns exit 1 when count is 0 — wrap in `|| echo 0`
