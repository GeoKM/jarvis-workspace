### ZPM3 path and directory model

- Command path resolved in ZSHOW as:
  - symbolic: `$0 A0 B0 J1 J2`
  - DU form: `A0 A0 B0 J1 J2`
  - DIR form: `COMMANDS COMMANDS WORK WS F80`
- Confirmed named directories:
  - `A0` = `COMMANDS`
  - `B0` = `WORK`
  - `J1` = `WS`
  - `J2` = `F80`
  - `J3` = `TP1`
  - `J4` = `TP3`
  - `J5` = `SCALC`
  - `J6` = `BASIC`
  - `J7` = `MULTIPLN`
  - `J8` = `DBASEII`
  - `K0` = `GAMES`
  - `K1` = `ZORK`
- System file names from ZSHOW:
  - shell variable file = `SH.VAR`
  - system file names 1-4 undefined
- Environment data from ZSHOW:
  - console width `80`, total lines `24`, text lines `22`
  - LST width `80`, total lines `66`, text lines `58`, formfeed `YES`
  - `DU OK = YES`, `Max DU = P15`, `Speed = 4 MHz`, `Quiet = NO`
  - drive map `A..P`

### ZPM3 operating notes

- `DIR` does not accept named directory or DU targets as an argument on this build (for example `DIR WS:` and `DIR J1:` returned `ERROR: Illegal command tail.`).
- Correct navigation pattern is to change into a named directory first (for example `WS:` / `TP1:`), then run `DIR` from inside that area.
- `DIRS` appears to be an interactive command on this system rather than a plain one-shot directory listing command.
